# ccm v1.3 — the split-flap board (interface design spec)

Date: 2026-07-06 · Status: implemented; palette validated with the dataviz checker

## Concept

ccm's subject is time and boarding: quota windows reset on schedules, accounts get
boarded (launched), sessions transfer between accounts. So the interface is a
**Solari split-flap departure board**, shipped in two mediums sharing one design
language: a full-screen zero-dependency TUI (`ccm` / `ccm pick`) and a local web page
(`ccm ui`, `src/ui/board.html` served by `src/ui/server.js`).

**Signature element**: the split-flap cascade — cells scramble through characters and
settle left-to-right on load/refresh (per-cell char cycling + canvas scrambler hook in
the TUI; char cycling + rotateX pulse per cell on the web). One bold move; everything
else stays quiet. `prefers-reduced-motion` disables it on the web.

## Tokens (src/tui/theme.js — shared by both mediums)

Board #0D0D0D · tile #1A1A19 · ink #F5F3EA (warm paper white) · secondary #C3C2B7 ·
muted #898781 · signal amber #FAB219 (time, headers — on a departure board, time IS
the warning; the accent/warning double duty is deliberate) · status good #0CA30C /
serious #EC835A / critical #D03B3B. Account "line colors" map registry names to the
dark-surface categorical palette: cyan→#3987E5, magenta→#D55181, yellow→#C98500,
green→#199E70, blue→#9085E9, red→#E66767 — validated (all ≥3:1 on #1A1A19, worst
adjacent CVD ΔE 41.3). Dark-committed: departure boards have no light mode.
Type: board text is monospace (Cascadia Mono — Windows Terminal's own face; every
character sits in a flap cell); supporting copy on the web uses Segoe UI.

## TUI (src/tui/)

- `term.js`: alt-screen + raw-key + resize lifecycle; `Canvas` cell grid serialized
  to one ANSI write per frame (truecolor, no flicker, honors NO_COLOR).
- `flap.js`: `Cascade` (left→right settle sweep with jitter), the seam-split CCM
  wordmark, tile-meter drawing (8 tiles of '██'/'··' with gap — the 2px spacer rule).
- `app.js`: state machine with three views — **board** (rows sorted most-headroom
  first, chips: IN SESSION / ✦ MOST HEADROOM / ▲ ALMOST FULL / ■ FULL, live station
  clock, 1–9 quick-board), **departures** (sessions for cwd across all accounts with
  best-effort titles from transcript heads; enter resumes, m opens the transfer
  overlay), **doctor** (structured lamps from collectDoctor). Launching leaves the
  alt screen, runs Claude Code inline, and returns to the board when the session ends.
  Renderers are pure (state, w, h) → Canvas, so they're headless-tested.

## Web (src/ui/)

Zero-dep http server bound to 127.0.0.1 with a Host allowlist (anti-DNS-rebinding),
JSON API (`/api/state`, `/api/doctor`, POST `refresh`/`launch`/`move`), single
self-contained page. "Board" opens a Windows Terminal tab running `ccm <name>`
(wt.exe, cmd fallback); transfers copy the session then open the target account
resumed. Meters are flap-tile rows (status color only at ≥80/95, icon+label chips so
color never carries meaning alone); focus outlines visible; toast for action feedback;
20s polling plus refresh-on-focus.

## Retired

The v1.0 minimal arrow-list picker (picker.js now only exports sortByHeadroom).
